# Render 배포 가이드 (Deployment Guide)

이 문서는 **MyStaGram** 프로젝트를 무료 클라우드 플랫폼인 **Render (render.com)**에 배포하는 방법을 안내합니다.

## 1. 전제 조건
*   **GitHub 계정**: 코드가 GitHub 저장소(Repository)에 올라가 있어야 합니다.
*   **Render 계정**: [dashboard.render.com](https://dashboard.render.com)에 가입되어 있어야 합니다. (GitHub 계정으로 가입 추천)

## 2. 프로젝트 설정 파일 확인
프로젝트 루트에 `render.yaml` 파일이 이미 생성되어 있습니다. 이 파일은 Render에게 "어떻게 빌드하고 실행할지"를 알려줍니다.

**`render.yaml` 내용:**
```yaml
services:
  - type: web
    name: mystagram
    runtime: python
    # 1. uv 설치 및 의존성 동기화
    buildCommand: pip install uv && uv sync
    # 2. 서버 실행 (uv run 사용)
    startCommand: uv run uvicorn app.main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: PYTHON_VERSION
        value: 3.12.0
```

## 3. 배포 단계

### 3.1. GitHub에 코드 푸시(Push)
아직 코드를 올리지 않았다면, GitHub에 새 레포지토리를 만들고 코드를 업로드하세요.

### 3.2. Render에서 'Blueprint' 생성
1.  Render 대시보드에서 **[New +]** 버튼 클릭 -> **[Blueprint]** 선택
2.  **Connect a repository** 목록에서 MyStaGram 리포지토리 선택 (보이지 않으면 `Configure account` 눌러서 권한 허용)
3.  **Service Name** 입력 (예: `mystagram-workshop`)
4.  **Apply** 클릭

### 3.3. 배포 확인
*   Render가 `render.yaml`을 읽어 자동으로 서버를 세팅합니다.
*   Logs 탭에서 빌드 과정을 볼 수 있습니다.
*   "Live" 상태가 되면 제공된 URL(예: `https://mystagram-workshop.onrender.com`)로 접속해봅니다.

## 4. 주의사항 (SQLite 데이터 초기화)
Render의 무료 플랜(Free Tier)은 **파일 시스템이 영구적이지 않습니다.**
*   **현상**: 서버를 새로 배포하거나, 일정 시간 접속이 없어 서버가 잠들었다 깨어나면 `data/insta.db` 파일이 초기 상태로 돌아갑니다.
*   **해결책**: 데이터를 영구 보관하려면 Render의 **PostgreSQL** 서비스를 추가로 연동해야 합니다. (본 워크숍 범위를 넘어섬)
