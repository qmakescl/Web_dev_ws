# API 주도 개발 (API-Driven Development)과 FastAPI

이 문서는 **MyStaGram** 프로젝트가 채택한 **API 주도 개발** 방식의 목적과 효율성을 설명하고, 이를 구현하기 위해 사용된 **FastAPI** 프레임워크와 실제 구현된 API들을 소개합니다.

## 1. 왜 API를 사용하는가?

과거의 웹 개발은 서버가 HTML(화면)을 통째로 만들어 브라우저에게 던져주는 방식(Server-Side Rendering)이 주를 이루었습니다. 하지만 모바일 앱, 스마트 기기 등 다양한 클라이언트가 등장하면서 **데이터만 주고받는 방식**의 필요성이 대두되었습니다.

### 1.1. API 사용의 목적
*   **분리 (Decoupling)**: 프론트엔드(화면)와 백엔드(데이터/로직)를 분리하여 서로 독립적으로 개발하고 배포할 수 있습니다.
*   **재사용성 (Reusability)**: 하나의 백엔드 API를 웹, 아이폰 앱, 안드로이드 앱 등 여러 곳에서 공통으로 사용할 수 있습니다.
*   **유연성 (Flexibility)**: 화면 디자인을 바꿀 때 백엔드 코드를 건드릴 필요가 없고, 반대로 DB를 교체해도 화면 코드는 영향받지 않습니다.

### 1.2. 효율성
*   **데이터 전송량 감소**: HTML 전체를 매번 전송하는 대신, 필요한 JSON 데이터만 주고받으므로 네트워크 비용이 절약됩니다.
*   **사용자 경험 향상**: 페이지 전체를 새로고침하지 않고(AJAX/Fetch), 필요한 부분만 부드럽게 업데이트할 수 있습니다.

## 2. FastAPI: 현대적인 파이썬 웹 프레임워크

이 프로젝트는 **FastAPI**를 사용하여 API 서버를 구축했습니다.

### 2.1. FastAPI 선택 이유
1.  **빠른 속도**: NodeJS나 Go에 필적하는 매우 높은 성능을 자랑합니다.
2.  **자동 문서화**: 코드를 짜면 Swagger UI(`/docs`)가 자동으로 생성되어, 별도로 API 문서를 작성할 필요가 없습니다.
3.  **타입 안전성**: Python의 Type Hint를 적극 활용하여, 데이터 검증을 자동으로 수행하고 버그를 줄여줍니다.
4.  **생산성**: 코드가 간결하고 직관적이어서 개발 속도가 빠릅니다.

## 3. MyStaGram의 API 구조

이 워크스페이스에서는 기능별로 **Router**를 나누어 API를 관리하고 있습니다.

### 3.1. 인증 (Auth) - `app/routers/auth.py`
사용자의 신원을 확인하고 출입증(Token)을 발급합니다.
*   `POST /api/auth/register`: 회원가입 (사용자 생성)
*   `POST /api/auth/login`: 로그인 (JWT 토큰 발급)

### 3.2. 게시물 (Posts) - `app/routers/posts.py`
SNS의 핵심인 게시물을 관리합니다.
*   `GET /api/posts`: 전체 피드 조회
*   `POST /api/posts`: 게시물 작성 (이미지 업로드)
*   `GET /api/posts/{id}`: 게시물 상세 조회
*   `PUT /api/posts/{id}`: 게시물 수정
*   `DELETE /api/posts/{id}`: 게시물 삭제

### 3.3. 상호작용 (Interactions)
사용자 간의 소통을 담당합니다.
*   **댓글** (`app/routers/comments.py`):
    *   `POST /api/posts/{id}/comments`: 댓글 달기
    *   `GET /api/posts/{id}/comments`: 댓글 목록 보기
*   **좋아요** (`app/routers/likes.py`):
    *   `POST /api/posts/{id}/like`: 좋아요 누르기/취소하기

### 3.4. 태그 검색 (Tags) - `app/routers/tags.py`
해시태그를 통한 검색 기능을 제공합니다.
*   `GET /api/tags/{tag_name}`: 특정 태그가 달린 게시물들만 필터링하여 조회
